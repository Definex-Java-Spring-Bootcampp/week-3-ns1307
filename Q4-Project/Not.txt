Projeyi yetiştiremedim ancak kafamdaki şey şuydu:
Her servis birbirini tetikleyecek ve API üzerinden birbirlerinde obje oluşturacaklar.

customer-service, yeni müşteri oluştuurulduğunda müşteri objesi oluşturur. 
order-service, Order içerisinde OrderItem listi vardır. OrderItem ürünü ve miktarını tutar.Bu sayede Order'a birden fazla ürün eklenebilir. order-service'de müşterinin email'ini customerID gibi tutarak bir order objesi oluştulur. Her bir order invoice'a sahiptir. invoice-service'den yeni bir invoiceNo alınır.
product-service, ürün ile ilgili bilgileri içerir. order-service'deki orderItem'de product vardır.
invoice-service,  siparişin faturasıdır. invoiceNo bulundurur ve unique bir invoiceNo vermekten sorumludur.

main-service, bütün bu işlemlerin ortak olarak yönetildiği sınıftır. Bu sürecin yönetimini sağlar. İletişim API üzerinden objeler yaratarak gerçekleştirilir.



Şu an için projede projectDiscovery'de Eureka server oluşturdum ve tüm servislerin server'a kaydını yaptım. 
ProjectGateway, gateway işlemlerini halleder, kurulumu tamamlandı. Servisler'e api endpoint eklendikçe, routing işlemi tamamlanacak.
GlobalExceptionHandler main-service'de kuruldu, verilen hatalarla bu kısım ilgilenecek.

Sadece mikroservislerin API iletişimi için API methodlarının oluşturulması ve birbiriyle iletişime geçerek objelerinin oluşturulması aşaması kaldı.
